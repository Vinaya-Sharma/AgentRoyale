Contamination-proof

Ground truth is fetched live before every evaluation. Therefore answers cannot be memorized or pre-cached and there is no answer key to exploit.

A 2026 UC Berkeley audit found all 8 leading agent benchmarks, including SWE-bench, WebArena, and GAIA, could be exploited for near-perfect scores without solving a single task. Static answer keys make this trivial. Agent Royale reduces several common benchmark exploit paths by using live, time-varying answers.

◎

Retrieval, not recall

Most benchmarks cannot tell the difference between a model remembering a fact from training and actually fetching it live, but retrieval reliability failures are costly.

Most production AI agents fail because of stale values, wrong sources, wrong regions, or wrong product variants.

Agent Royale specifically tests: which agent stack can reliably retrieve the public web today?

🎯

What Agent Royale measures

Agent Royale evaluates live retrieval reliability end-to-end.

  • Does the agent find the right source?
  • Does it extract the correct current value?
  • Does it avoid nearby wrong values?
  • Does its citation support the answer?
  • How much does it cost?
  • How long does it take?
  • How consistent is it across repeated runs?
!

AI has gotten very good at looking right. Search agents cite real pages, write polished answers, and still return the wrong value. GhostCite benchmarked 13 leading LLMs across 40 domains and found hallucination rates ranging from 14% to 95% even when models cited a real source. The models were not failing to search. They were searching, citing, and still getting it wrong more than half the time. Agent Royale exists to warn that we cannot trust AI just yet, and to help measure whether we are getting closer to the point where we can.

Why existing benchmarks are not enough
Search agents now cite sources confidently, but citation does not equal accuracy.
Every AI product is becoming an agent, but agents still fail at basic live facts: prices, ratings, versions, counts, availability, and policy limits. Existing benchmarks are often static, leaked, or gameable. Agent Royale creates a harder-to-game measurement layer for live web retrieval: current questions, independent ground truth, exact grading, citation verification, latency, and cost. It gives teams a practical answer to: which agent stack can reliably retrieve the public web today?

The Arena

Compare two anonymous correct answers and choose the route that felt more direct, reliable, and human.

How to vote

Pick the path that looks more direct, reliable, and source-appropriate. Do not simply vote for fewer citations.

These guidelines are only direction. At the end of the day, vote with your human intuition: which route felt more optimal for getting to the answer?

  • Better: canonical source used early, relevant URLs, clear extraction.
  • Worse: stale articles, third-party distractors, wrong region/SKU, source hopping.
  • Tie: both paths use similarly trustworthy sources with similar focus.

Example

Better: Apple official buy page → base MacBook Air price.

Worse: Apple page plus Walmart, blogs, old newsroom posts, or education pricing for the same answer.

An ideal route is the route a careful human would naturally take for this task.

Agent Royale evaluates one primitive: can an AI stack retrieve a specific current value from the live web and return the exact answer?

What This System Measures

Live Exact Accuracy is a pass/fail grade against live ground truth fetched from the task's canonical source. No LLM judge decides whether an answer is "good enough"; the extracted claim either matches after normalization and tolerance or it does not.

Verified Retrieval Rate measures whether correct answers were backed by a cited source fetch during the run. A correct answer without a matching citation is not treated as verified retrieval.

Path quality is kept separate. Humans vote between correct traces only, so a fast path to the wrong answer is never rewarded.

MVP Runner

The current implementation uses your existing OpenRouter and Bright Data keys. Bright Data fetches the canonical URL only for ground truth. OpenRouter models answer using the openrouter:web_search server tool, configured for native provider search where available.

Bright Data canonical fetch -> ground truth
OpenRouter model + web_search -> model answer + citations
claim extraction -> exact/numeric/currency grading -> stored run -> leaderboard

OpenRouter uses native provider search for supported providers and can fall back depending on model/provider support. The run trace stores cited URLs and search-request counts returned by OpenRouter.

Arena vs Live Check

Arena is for blind pairwise voting on path quality. It samples stored benchmark traces where both models got the answer right, so votes measure retrieval efficiency rather than accuracy.

Live Check is an ad hoc verification tool. It runs one task now with fresh ground truth and stores the result separately in live_checks.jsonl, so demos and probes never change the official leaderboard.
