The Arena

How to vote

Pick the path that looks more direct, reliable, and source-appropriate. Do not simply vote for fewer citations.

These guidelines are only direction. At the end of the day, vote with your human intuition: which route felt more optimal for getting to the answer?

  • Better: canonical source used early, relevant URLs, clear extraction.
  • Worse: stale articles, third-party distractors, wrong region/SKU, source hopping.
  • Tie: both paths use similarly trustworthy sources with similar focus.

Example

Better: Apple official buy page → base MacBook Air price.

Worse: Apple page plus Walmart, blogs, old newsroom posts, or education pricing for the same answer.

An ideal route is the route a careful human would naturally take for this task.

Agent Royale evaluates one primitive: can an AI stack retrieve a specific current value from the live web and return the exact answer?

What This System Measures

Live Exact Accuracy is a pass/fail grade against live ground truth fetched from the task's canonical source. No LLM judge decides whether an answer is "good enough"; the extracted claim either matches after normalization and tolerance or it does not.

Verified Retrieval Rate measures whether correct answers were backed by a cited source fetch during the run. A correct answer without a matching citation is not treated as verified retrieval.

Path quality is kept separate. Humans vote between correct traces only, so a fast path to the wrong answer is never rewarded.

MVP Runner

The current implementation uses your existing OpenRouter and Bright Data keys. Bright Data fetches the canonical URL only for ground truth. OpenRouter models answer using the openrouter:web_search server tool, configured for native provider search where available.

Bright Data canonical fetch -> ground truth
OpenRouter model + web_search -> model answer + citations
claim extraction -> exact/numeric/currency grading -> stored run -> leaderboard

OpenRouter uses native provider search for supported providers and can fall back depending on model/provider support. The run trace stores cited URLs and search-request counts returned by OpenRouter.

Arena vs Live Check

Arena is for blind pairwise voting on path quality. It samples stored benchmark traces where both models got the answer right, so votes measure retrieval efficiency rather than accuracy.

Live Check is an ad hoc verification tool. It runs one task now with fresh ground truth and stores the result separately in live_checks.jsonl, so demos and probes never change the official leaderboard.
